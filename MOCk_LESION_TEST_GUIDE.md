# Mock 病灶显像功能 - 测试验证指南

## 📋 功能概述

本功能通过在后端随机生成模拟病灶数据，在前端 Canvas 上以半透明红色椭圆叠加层的形式渲染，实现"病灶显像"的演示效果。

---

## 🔧 部署步骤

### 1. 执行数据库迁移

在 MySQL 中执行以下 SQL 脚本：

```bash
mysql -u root -p med_viewer_db < database_migration_add_mock_lesion.sql
```

或者手动执行 `database_migration_add_mock_lesion.sql` 中的 SQL 语句。

**验证字段是否添加成功：**
```sql
DESCRIBE med_image;
-- 应该看到 has_mock_lesion (TINYINT) 和 mock_lesion_data (TEXT) 两个新字段
```

### 2. 重启后端服务

```bash
cd D:\bishe\Graduation_Project
mvn clean package
mvn spring-boot:run
```

### 3. 启动 Python 服务（如果未运行）

```bash
cd Python_script
python image_parser_service.py
```

### 4. 访问前端

浏览器打开：`http://localhost:8080/index.html`

## 🧪 测试流程

### 测试场景 1：上传影像并验证 Mock 数据生成

1. **登录系统**
   - 使用任意账号登录（需已加入课题组）

2. **上传医学影像**
   - 进入"影像管理"模块
   - 点击"上传影像"按钮
   - 选择一个 DICOM/NIfTI 文件
   - 选择可见性（本组私有/公开共享）
   - 确认上传

3. **检查数据库**
   ```sql
   SELECT id, file_name, has_mock_lesion, mock_lesion_data 
   FROM med_image 
   ORDER BY create_time DESC 
   LIMIT 5;
   ```
   
   **预期结果：**
   - 约 50% 的记录 `has_mock_lesion = 1`
   - 有病灶的记录 `mock_lesion_data` 包含 JSON 格式的坐标数据：
     ```json
     {"centerX":128.50,"centerY":135.20,"radiusX":25.60,"radiusY":22.40,"type":"ellipse"}
     ```

### 测试场景 3：前端病灶显像 - 分支 B（有病灶）

1. **进入分析模块**
   - 在"影像管理"列表中找到一个 `has_mock_lesion = 1` 的影像
   - 点击"查看"图标

2. **等待影像加载完成**
   - 确保切片图像正常显示
   - "病灶显像"按钮变为可用状态

3. **开启病灶显像**
   - 点击"病灶显像"按钮
   
4. **预期结果：**
   - ✅ 弹出绿色 Toast 提示："已开启病灶显像"
   - ✅ 按钮文字变为"隐藏病灶"，边框变为红色
   - ✅ 影像上出现半透明红色椭圆区域（rgba(255, 0, 0, 0.4)）
   - ✅ 椭圆边缘有轻微模糊效果（4px 高斯模糊）
   - ✅ 椭圆中心有一个白色标记点
   - ✅ 椭圆位置与影像尺寸成比例（基于 mock_lesion_data 中的坐标）

5. **关闭病灶显像**
   - 再次点击"隐藏病灶"按钮
   
6. **预期结果：**
   - ✅ 弹出蓝色 Toast 提示："已关闭病灶显像"
   - ✅ 按钮文字恢复为"病灶显像"，边框恢复灰色
   - ✅ 红色椭圆叠加层消失，原始影像完全可见

### 测试场景 4：切换切片时病灶保持

1. **在有病灶的影像分析页面**
   - 开启病灶显像

2. **拖动底部滑块切换切片**
   - 观察病灶叠加层

3. **预期结果：**
   - ⚠️ **当前实现**：病灶会随切片刷新而清除（因为每次 loadSlice 都会重新加载图片）

## 🐛 常见问题排查

### 问题 1：上传后数据库中没有 Mock 数据

**原因：** 后端 `generateMockLesionData()` 方法未被调用

**解决：**
- 检查 `ImageService.handleUpload()` 中是否调用了 `generateMockLesionData(medImage)`
- 查看后端日志是否有异常

---

### 问题 2：前端点击按钮无反应

**原因：** `currentImageMeta` 未正确传递

**解决：**
- 打开浏览器开发者工具（F12）
- 在 Console 中检查是否有 JavaScript 错误
- 确认 `gotoAnalysis()` 第三个参数是否正确传入影像对象

---

### 问题 3：Canvas 不显示或位置偏移

**原因：** Canvas 尺寸与图片不一致

**解决：**
- 检查 `renderLesionOverlay()` 中是否正确设置了 `canvas.width/height`
- 确认 CSS 中 Canvas 使用了 `position: absolute` 且 `inset: 0`

---

### 问题 4：病灶坐标不准确

**原因：** 缩放比例计算错误

**解决：**
- 检查 `scaleX = canvas.width / currentImageMeta.dimX` 的计算
- 确认 `dimX/dimY` 是原始影像尺寸，而非显示尺寸
